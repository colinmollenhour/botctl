# botctl docs

`botctl` helps you keep Claude Code sessions under control inside `tmux`.

The fun part is not the implementation details. The fun part is that you can:

- start one local `runtime` that owns observation and automation
- open a live `dashboard` and immediately see what all your Claude panes are doing
- turn on `yolo` and let `botctl` set shared babysitting policy for the boring confirmation flows
- run `serve` and get a runtime-backed event stream or HTTP facade when you want to watch or integrate the session externally

Also, yes, it is written in Rust. Legally, I believe, all new CLIs must be.

## The four cool commands

### `runtime`

The command that owns the live control plane.

- holds the authoritative pane snapshots in memory
- runs the shared observation and guarded action loop
- supervises yolo centrally instead of letting each client invent its own view
- listens on `<state-dir>/runtime.sock`

### `dashboard`

The command you open first.

- see Claude panes across workspaces in one place
- spot blocked or waiting sessions quickly
- jump straight into a pane
- toggle YOLO without hunting around in tmux

### `yolo`

The command that removes tedious operator babysitting.

- handle common safe confirmation flows automatically
- scope automation to one pane or one workspace
- keep the guardrails on instead of blindly mashing keys

### `serve`

The runtime-backed API and event facade.

- stream what is happening in a session as it happens
- use human output for watching, or JSONL for tooling
- expose a localhost HTTP API over shared runtime state

## Install

```bash
cargo install botctl
```

## Quick start

```bash
botctl runtime
botctl dashboard
botctl yolo --pane 0:6.0
botctl serve --session demo --format jsonl
```
