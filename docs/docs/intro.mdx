# botctl docs

`botctl` helps you keep Claude Code, Codex CLI, and OpenCode sessions visible and controlled inside `tmux`.

Claude Code panes get the full guarded automation path. Codex CLI panes are classified from captured terminal screens and can use YOLO for command permission approval. OpenCode panes are discovered passively for dashboard visibility, recent-message context, state classification, and tmux window status.

Use `botctl` when you want to:

- open a live `dashboard` and see what Claude Code, screen-detected Codex CLI, and resolvable OpenCode panes are doing
- turn on `yolo` for guarded permission approval
- run `serve` for a foreground event stream or localhost HTTP API
- export the latest persisted assistant reply with `last-message`

## Provider support

| Provider | Dashboard | Classification | YOLO | Prompt submission | `last-message` |
| --- | --- | --- | --- | --- | --- |
| Claude Code | yes | yes | yes | yes | yes |
| Codex CLI | yes, from screen capture | yes, from screen capture | command permission approval only | no | yes |
| OpenCode | yes, after safe title plus cwd resolution | limited passive state from SQLite | no | no | yes |

## Main commands

### `dashboard`

Use this first when you have multiple tmux panes open.

- Shows Claude Code panes, screen-detected Codex CLI panes, and passively resolvable OpenCode panes across workspaces.
- Shows state, age, branch, recent context, and safe actions.
- Lets you jump into a pane and toggle YOLO for Claude Code and Codex panes.
- Can run persistently inside a tmux popup.

### `yolo`

Use this when a pane is safe to babysit.

- Acts only after classification finds a supported state.
- Scopes automation to one pane, all panes, or one workspace.
- Approves Claude permission dialogs, dismisses Claude surveys, and approves Codex command permission dialogs.
- Refuses ambiguous targets and unsupported states.

### `serve`

Use this when another tool needs live state.

- Streams human-readable or JSONL events for one tmux session.
- Can expose a localhost HTTP API while the foreground process runs.
- Persists event tapes and summaries under the botctl state root.

## Install

```bash
cargo install botctl
```

## Quick start

```bash
botctl dashboard
botctl yolo --pane 0:6.0
botctl serve --session demo --format jsonl
```
