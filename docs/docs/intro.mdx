# botctl docs

`botctl` helps you keep Claude Code, Codex CLI, and OpenCode sessions visible and controlled inside `tmux`.

Claude Code panes get the full guarded automation path. Codex CLI panes are classified from captured terminal screens and can use YOLO for command permission approval. OpenCode panes are discovered passively for dashboard visibility, recent-message context, state classification, and tmux window status.

The fun part is not the implementation details. The fun part is that you can:

- open a live `dashboard` and immediately see what your Claude Code, screen-detected Codex CLI, and resolvable OpenCode panes are doing
- turn on `yolo` and let `botctl` babysit the boring confirmation flows
- run `serve` and get a live event stream when you want to watch or integrate the session externally

Also, yes, it is written in Rust. Legally, I believe, all new CLIs must be.

## The three cool commands

### `dashboard`

The command you open first.

- see Claude Code panes, screen-detected Codex CLI panes, and passive OpenCode pane visibility across workspaces in one place
- spot blocked or waiting sessions quickly
- jump straight into a pane
- toggle YOLO without hunting around in tmux

### `yolo`

The command that removes tedious operator babysitting.

- handle common safe confirmation flows automatically
- scope automation to one pane or one workspace
- keep the guardrails on instead of blindly mashing keys

### `serve`

The command for live observation.

- stream what is happening in a session as it happens
- use human output for watching, or JSONL for tooling
- keep a long-running foreground observer attached to one tmux session

## Install

```bash
cargo install botctl
```

## Quick start

```bash
botctl dashboard
botctl yolo --pane 0:6.0
botctl serve --session demo --format jsonl
```
